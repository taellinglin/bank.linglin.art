Luna Coin Blockchain Tutorial

A comprehensive guide to the Luna Coin blockchain system

Overview

Luna Coin is a blockchain-based cryptocurrency system that uses a unique approach combining GTX Genesis transactions (representing physical or digital bills) with traditional blockchain features like transfers and mining rewards.

Core Components

1. Blockchain Structure

  • Blocks: Contain transactions, timestamps, hashes, and proof-of-work data
  • Genesis Block: The first block (index 0) that initializes the chain
  • Mempool: A pool of pending transactions waiting to be mined
  • Mined Indexes: Tracks which serial numbers have already been mined

2. Transaction Types

GTX Genesis Transactions

These represent the foundation of Luna Coin - digital bills with serial numbers

Transfer Transactions

Wallet-to-wallet transfers of Luna Coins

Reward Transactions

Mining rewards for validating blocks

How Luna Coin Works

1 System Initialization

When starting the blockchain daemon:

  • Creates/Loads Blockchain: Loads existing blockchain or creates genesis block
  • Initializes Mempool: Loads pending transactions
  • Bootstraps with Genesis Transactions: If no GTX transactions exist, creates initial set
  • Starts Background Services: Confirmation updater, auto-mining, cleanup

2 Adding Transactions

Adding Genesis Bills

blockchain.add_genesis_transaction(
    serial_number="SN-MYBILL-001",
    denomination=500.0,
    issued_to="Alice"
)

Adding Transfers

blockchain.add_transaction({
    "type": "transfer",
    "from": "Alice",
    "to": "Bob",
    "amount": 25.0,
    "timestamp": time.time(),
    "signature": "valid_signature_here"
})

The mining process:

  1. Selects transactions from mempool
  2. Creates a new block with those transactions
  3. Performs proof-of-work to find a valid hash
  4. Adds the validated block to the blockchain
  5. Removes mined transactions from mempool
  6. Creates reward transaction for the miner
# Mining a block
mined_block = blockchain.mine_pending_transactions(
    miner_address="miner_123",
    difficulty=4
)

4 Reward System

Miners receive rewards based on:

  • Base reward per bill: 1 Luna Coin per GTX Genesis transaction
  • Bill value multiplier: Higher denomination bills may increase difficulty
  • Block inclusion: Rewards are created as transactions in the next block
Note: The reward system incentivizes miners to include GTX Genesis transactions in blocks, which forms the economic foundation of Luna Coin.

Key Features

Double-Spending Protection

Luna Coin prevents double-spending through:

  • Serial number tracking for GTX transactions
  • Transaction hash verification
  • Blockchain validation

Dynamic Difficulty

Mining difficulty adjusts based on:

  • Bill denominations being mined
  • Number of transactions in block
  • Total value of transactions

Auto-Recovery

The system includes robust error handling:

  • Corrupted file detection and repair
  • Emergency blockchain rebuilding
  • Automatic backup creation

Usage Examples

Starting the Daemon

# Initialize blockchain
blockchain = BlockchainDaemon()

# Start background processes
blockchain.start_daemon(miner_address="my_miner")

Checking Status

# Get blockchain statistics
stats = blockchain.get_blockchain_stats()
print(f"Blocks: {stats['total_blocks']}")
print(f"Transactions: {stats['total_transactions']}")
print(f"Mempool: {stats['mempool_size']}")

Manual Mining

# Manually mine pending transactions
blockchain.mine_pending_transactions(
    miner_address="manual_miner",
    difficulty=4
)
